# 섹션-가사 반복 페이지 매핑 설계

## 목표

콘티 곡 편집의 섹션-가사 매핑에서 가사 페이지 라벨에 마우스를 올리면 가사 미리보기 버블이 다시 보이게 하고, 같은 가사 페이지를 한 섹션 안에 여러 번 배치할 수 있게 한다.

## 범위

- 섹션별 가사 페이지 선택 UI를 체크박스 토글에서 "추가" 동작으로 바꾼다.
- 사용자는 `페이지 1`, `페이지 2`, `페이지 1`처럼 동일 페이지를 반복 순서에 넣을 수 있다.
- 선택된 페이지는 순서를 보존하며 표시하고, 각 항목을 개별 삭제할 수 있다.
- 선택 라벨과 선택된 배지 모두 hover 시 해당 가사 페이지 내용을 Tooltip으로 보여준다.
- 기존 저장 형식 `sectionLyricsMap: Record<number, number[]>`는 유지한다. 배열에 중복 인덱스를 허용하는 방식으로 반복을 표현한다.

## UI 동작

- 각 섹션 accordion 안에는 모든 가사 페이지 버튼을 표시한다.
- 버튼 클릭 시 해당 페이지 인덱스를 현재 섹션의 배열 끝에 추가한다.
- 섹션 제목의 페이지 수는 중복을 포함한 총 배치 수를 보여준다.
- 섹션 안에 "선택된 순서" 영역을 두고, 선택된 페이지 배지를 순서대로 보여준다.
- 선택된 배지의 삭제 버튼을 누르면 해당 위치의 배치만 제거한다.
- 미리보기 영역도 동일한 순서와 중복을 그대로 보여준다.

## 데이터 흐름

- `SectionLyricsMapper`가 기존 `initialMap`을 받아 내부 상태로 관리한다.
- 추가/삭제/정리 로직은 작은 순수 함수로 분리해 테스트한다.
- 가사 페이지 삭제 등으로 유효하지 않은 인덱스가 생기면 기존처럼 정리하되, 유효한 중복 인덱스는 제거하지 않는다.
- 저장 액션과 PPT export는 기존 배열 순서를 그대로 사용하므로 DB 마이그레이션은 필요 없다.

## 오류 처리

- 가사 페이지가 없으면 기존 안내 문구를 유지한다.
- 빈 가사 페이지 Tooltip은 `(빈 페이지)`로 표시한다.
- 잘못된 페이지 인덱스는 가사 페이지 수 기준 정리 단계에서 제거한다.

## 검증

- 순수 함수 테스트로 같은 페이지를 여러 번 추가할 수 있고, 특정 위치만 삭제되는지 확인한다.
- 유효하지 않은 페이지 인덱스 정리 시 중복 유효 인덱스가 유지되는지 확인한다.
- 소스 테스트로 Tooltip trigger/content와 추가/삭제 UI 문구가 남아 있는지 확인한다.
- 브라우저에서 콘티 곡 편집 drawer를 열어 hover 미리보기와 반복 추가 동작을 확인한다.
